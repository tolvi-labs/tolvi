## [14:30] Session — Landed the Postgres-over-MySQL decision

Spent the afternoon walking through the comparison matrix one more time with the platform team. The JSONB indexing demo on the event table was the moment everyone agreed: the same query that took 1.4s on a MySQL prototype dropped to 60ms on Postgres with a single GIN index. Wrote up the choice as [[choose-postgres-over-mysql]] and circulated the link. Filed a follow-up to add `pg_stat_statements` to the bootstrap and to draft a backup runbook. Tomorrow: PgBouncer config and a smoke test of logical replication into the staging search index.
