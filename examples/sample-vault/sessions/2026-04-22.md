## [09:15] Session — gRPC investigation and protocol buffers layout

Worked through the service-to-service migration plan. The shared `proto/` directory layout is the contentious part: do we co-locate `.proto` files with their owning service, or pull them all into a single repository? Settled on the latter for v1 — generation is simpler and cross-service version coordination is easier when everything builds from one source. Two prototype services now talk over gRPC end-to-end with reflection enabled in dev. Wrote up the choice as [[grpc-for-service-to-service]]. The 12ms latency improvement at the median was a nice side effect; the real win is catching field renames at compile time.

## [15:45] Session — Volta pin spike

Spiked Volta on a smaller repo to validate the workflow. Installation was painless, and the `volta` block in `package.json` did exactly what the documentation promised: `cd` into the repo and the right Node binary is on PATH within milliseconds. Confirmed the CI bootstrap works on the runner image we use. Sketched out the case for pinning Node with Volta and the pilot plan for next week; will write it up as a proper decision doc once we have results from two repos. The only friction was around developers who use a different version manager for unrelated projects, but Volta's PATH handling does not interfere with those tools when you leave the pinned repo.
